Overview

Open Code Review (short ocr) is an AI-powered code review CLI tool.

✔ Supports workspace changes, branch diff, and single-commit review modes
Supports LLM services with Anthropic and OpenAI-compatible protocols
Concurrent review with customizable timeouts
Output format supports text and json
Zero config for Claude Code users
WebUI viewer for visualizing review results

Install

Install

npm i -g @alibaba-group/open-code-review

Verify Installation

ocr version

Configuration & Verification

Manage CLI configuration settings, stored in ~/.opencodereview/config.json.

Interactive Setup (Recommended)

The easiest way to configure Open Code Review. Select from built-in providers or add a custom one.

ocr config provider

Model Selection

After choosing a provider, pick the model to use.

ocr config model

List Built-in Providers

View all available built-in LLM providers.

ocr llm providers

Manual Configuration

Command

ocr config set <key> <value>

Example

ocr config set llm.url https://api.anthropic.com \
    && ocr config set llm.auth_token {{your-api-key}} \
    && ocr config set llm.model claude-opus-4-6 \
    && ocr config set llm.use_anthropic true  \
    && ocr config set language Chinese

Supported Config Keys

llm.urlAPI URL

llm.auth_tokenAPI Key

llm.modelModel Name

llm.use_anthropicUse Anthropic SDK

telemetry.enabledTelemetry

languageOutput Language

llm.extra_bodyVendor-specific request body fields (JSON)

Verify Configuration

# Test LLM connection
ocr llm test

Run this command after configuration to test the LLM connection and confirm settings are correct.

ocr review

ocr review is the core command for initiating AI code review.

Review Modes

Workspace Mode (Default)

Reviews staged + unstaged + untracked changes in the current workspace.

ocr review

Branch Diff Mode

Reviews the diff between two refs (based on merge-base).

ocr review --from master --to dev-ref

Single Commit Mode

Reviews a specific commit (diff against its parent).

ocr review -c abc123

Advanced Usage

Review with Requirement Context

Provide the requirement context (including important boundaries) to improve review quality.

ocr review --background "requirement context"
ocr review -b "requirement context"
JSON Output

Output results in structured JSON format for programmatic processing.

ocr review --format json
ocr review -f json
Agent Mode

Output only the final summary without execution process, suitable for automation pipelines.

ocr review --audience agent

Claude Code Integration

If you are already a Claude Code user with the following environment variables configured, Open Code Review will recognize them automatically — no extra configuration needed:

export ANTHROPIC_BASE_URL=https://api.anthropic.com
export ANTHROPIC_AUTH_TOKEN=sk-ant-xxxxx
export ANTHROPIC_MODEL=claude-opus-4-6

# Open Code Review auto-detects these variables ✨

You can also use ocr config to override or supplement these settings.
